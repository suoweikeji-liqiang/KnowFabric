一、数据库表结构草案（SQL 风格）
下面这版是首期可落地版本，目标不是一次把所有未来字段都做满，而是先把：


文档接入


页面拆分


chunk 检索

这几件事立住。
我这里用偏 PostgreSQL 的写法表达，你后面即使用别的库，结构也能迁过去。

1. document
SQLCREATE TABLE document (    id BIGSERIAL PRIMARY KEY,    doc_id VARCHAR(64) NOT NULL UNIQUE,    domain_code VARCHAR(32) NOT NULL,    brand_code VARCHAR(64),    equipment_type_code VARCHAR(64),    document_type_code VARCHAR(64),    title TEXT,    file_name TEXT NOT NULL,    file_ext VARCHAR(16) NOT NULL,    mime_type VARCHAR(128),    original_path TEXT NOT NULL,    storage_path TEXT NOT NULL,    file_hash VARCHAR(128) NOT NULL,    file_size_bytes BIGINT NOT NULL,    page_count INT,    language_code VARCHAR(16),    source_batch_id VARCHAR(64),    source_domain VARCHAR(64),    import_time TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,    modified_time TIMESTAMP,    created_time TIMESTAMP,    doc_version INT NOT NULL DEFAULT 1,    parse_status VARCHAR(32) NOT NULL DEFAULT 'pending',    trust_level VARCHAR(8),    is_active BOOLEAN NOT NULL DEFAULT TRUE,    is_deleted BOOLEAN NOT NULL DEFAULT FALSE,    extra_meta JSONB NOT NULL DEFAULT '{}'::jsonb);CREATE INDEX idx_document_domain_code ON document(domain_code);CREATE INDEX idx_document_brand_code ON document(brand_code);CREATE INDEX idx_document_equipment_type_code ON document(equipment_type_code);CREATE INDEX idx_document_document_type_code ON document(document_type_code);CREATE INDEX idx_document_file_hash ON document(file_hash);CREATE INDEX idx_document_parse_status ON document(parse_status);

2. document_version
SQLCREATE TABLE document_version (    id BIGSERIAL PRIMARY KEY,    doc_id VARCHAR(64) NOT NULL,    version_no INT NOT NULL,    file_hash VARCHAR(128) NOT NULL,    storage_path TEXT NOT NULL,    change_reason TEXT,    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,    created_by VARCHAR(64),    UNIQUE (doc_id, version_no));CREATE INDEX idx_document_version_doc_id ON document_version(doc_id);

3. document_page
SQLCREATE TABLE document_page (    id BIGSERIAL PRIMARY KEY,    page_id VARCHAR(64) NOT NULL UNIQUE,    doc_id VARCHAR(64) NOT NULL,    page_no INT NOT NULL,    page_type VARCHAR(32),    raw_text TEXT,    cleaned_text TEXT,    page_summary TEXT,    page_visual_summary TEXT,    has_table BOOLEAN NOT NULL DEFAULT FALSE,    has_diagram BOOLEAN NOT NULL DEFAULT FALSE,    has_fault_code BOOLEAN NOT NULL DEFAULT FALSE,    has_procedure BOOLEAN NOT NULL DEFAULT FALSE,    page_image_path TEXT,    page_image_hash VARCHAR(128),    ocr_status VARCHAR(32) NOT NULL DEFAULT 'pending',    multimodal_status VARCHAR(32) NOT NULL DEFAULT 'pending',    confidence_score NUMERIC(5,4),    extra_meta JSONB NOT NULL DEFAULT '{}'::jsonb,    UNIQUE (doc_id, page_no));CREATE INDEX idx_document_page_doc_id ON document_page(doc_id);CREATE INDEX idx_document_page_page_no ON document_page(page_no);CREATE INDEX idx_document_page_has_table ON document_page(has_table);CREATE INDEX idx_document_page_has_diagram ON document_page(has_diagram);

4. page_asset
SQLCREATE TABLE page_asset (    id BIGSERIAL PRIMARY KEY,    asset_id VARCHAR(64) NOT NULL UNIQUE,    page_id VARCHAR(64) NOT NULL,    doc_id VARCHAR(64) NOT NULL,    asset_type VARCHAR(32) NOT NULL, -- image/table/figure/thumbnail    asset_path TEXT NOT NULL,    asset_hash VARCHAR(128),    bbox_json JSONB,    extracted_text TEXT,    table_json JSONB,    extra_meta JSONB NOT NULL DEFAULT '{}'::jsonb);CREATE INDEX idx_page_asset_page_id ON page_asset(page_id);CREATE INDEX idx_page_asset_doc_id ON page_asset(doc_id);CREATE INDEX idx_page_asset_asset_type ON page_asset(asset_type);

15. 几个实现提醒
这套表结构的关键点是：


document / document_page / content_chunk / extracted_fact 是主链


review_record / processing_job / processing_stage_log 是工程可控性


domain_package 是未来扩展能力


向量本身不一定放数据库里，但至少要有 chunk_embedding_meta


也就是说，数据库里一定要存“向量的索引映射和追溯元数据”，不一定非要存浮点数组本体。
