# NexO&M 系统 UI 设计规范 (v1.0)

## 设计理念 (Design Philosophy)
本系统采用 **Linear / 现代高级 SaaS** 视觉语言。核心理念在于：极简、骨感、克制、专业。
通过极致的留白、极其克制的微底色与极细边框（Ring/Shadow），拉开清晰的 Z 轴空间层次，降低复杂 ToB 数据带来的视觉疲劳。

---

## 一、 Token 层 (Design Tokens)

Token 层是系统的视觉基石，所有颜色、间距、字号均来源于此。系统底层背景采用极浅的高级灰，使得纯白卡片能像悬浮在画布上一样清晰。

### 1. 颜色系统 (Color System)
- **品牌主色 (Primary)**: `#0244FD` (高保真科技蓝)
  - 衍生色阶：使用 Tailwind 的 `indigo` 色系平滑映射，从极浅的 50 到深沉的 950，保证交互状态（Hover、Active）的自然过渡。
- **系统背景 (Background)**: `#f8fafc` (slate-50) - 最干净的浅灰底色。
- **卡片背景 (Card)**: `#ffffff` (纯白) - 配合 `ring-1` 和微阴影拉开层次。
- **正文文本 (Foreground)**: `#0f172a` (slate-900) - 极深的岩石灰，避免纯黑带来的刺眼感。
- **辅助文本 (Muted)**: `#64748b` (slate-500) - 用于次要信息、表头、时间戳。
- **边界与分割线 (Border)**: `#e2e8f0` (slate-200) - 极细的分割线颜色。

### 2. 交互状态色 (Semantic Colors)
- **成功/运行 (Success)**: `#10b981` (emerald-500)
- **破坏性/严重故障 (Destructive)**: `#ef4444` (red-500)
- **警告/待处理 (Warning)**: `#f59e0b` (amber-500)
- **信息/进行中 (Info)**: `#3b82f6` (blue-500)

### 3. 基础变量 (Base Variables)
- **基础字号 (Font Size)**: `14px`
- **默认圆角 (Radius)**: `0.75rem` (12px) - 柔和现代的圆角处理。
- **聚焦光圈 (Ring)**: `#0244FD` (品牌主色，带 20% 透明度)

## 二、 语义层 (Semantic Layer)

语义层负责将抽象的设备状态、工单进度转化为直观的视觉表达。我们摒弃了粗糙的大色块，采用当前业界最流行的视觉方案。

### 1. 状态标签 (Status Badges)
核心规范：**“微浅底色 + 极细边框 (Ring) + 高亮圆点 / 同色系图标”**。视觉克制而不抢戏。

- **正常运行 (Success)**: `bg-emerald-50 text-emerald-700 ring-emerald-600/20`，配合绿色的 CheckCircle2。
- **设备故障 (Destructive)**: `bg-red-50 text-red-700 ring-red-600/20`，配合红色的 XCircle。
- **维保超期 (Warning)**: `bg-amber-50 text-amber-700 ring-amber-600/20`，配合琥珀色的 AlertTriangle。
- **网关离线 (Muted)**: `bg-slate-100 text-slate-600 ring-slate-500/20`，配合灰色的 WifiOff。
- **极端告警 (Critical)**: `bg-red-500 text-white shadow-red-500/30`（**唯一允许使用实心大色块的场景**，用于强制吸引注意力）。

## 三、 组件层 (Component Layer)

组件层定义了系统中高频复用的 UI 模块的视觉与交互细节。

### 1. 表格组件 (Data Table)
- **表头 (Table Header)**: 全大写英文字母 (Uppercase) + 拉宽字距 (Tracking-wider) + `12px` (text-xs) + `slate-500`，传达极致理性和硬核的专业感。
- **数据行 (Table Row)**: 行高不低于 `48px`。悬浮 (Hover) 时整行背景变为 `bg-slate-50/80`。
- **数据对齐**: 文本左对齐；数字、金额、核心指标（如温度、压力）**强制右对齐**并使用等宽字体 (Font-mono)。

### 2. 表单与输入框 (Form Inputs)
- **标签 (Label)**: 居于输入框上方，`14px`，加粗 `font-medium`。必填项后缀红色的 `*`。
- **输入框 (Input)**: 默认背景微灰 (`bg-slate-50/50`)，无边框感，视觉收敛。`Focus` 时背景变白并带有品牌色 `#0244FD` 的光晕。
- **错误校验**: 边框变红，输入框下方出现带 `XCircle` 小图标的红色提示文案。

### 3. 统计卡片 (Statistic Cards)
- **布局**: 主次字体对比强烈。指标名称使用辅助文本 (`14px`, `slate-500`)，绝对数值使用超大号字体 (`30px+`, `slate-900`, `tracking-tight`)，单位弱化缩小。
- **趋势指示**: 使用带微底色和箭头的标签表示同环比趋势（如红涨绿跌/绿涨红跌）。

### 4. 垂直时间轴 (Vertical Timeline)
- **骨架线**: 左侧带有一条贯穿的 `2px` 骨架线 (`bg-slate-100`)。
- **节点标记**: 最新状态在最上方。当前激活节点使用主色 `#0244FD` 并带呼吸环；已完成节点使用灰色；异常起点使用红色。

### 5. AI 助手气泡 (AI Assistant Panel)
- **风格**: 悬浮式对话框，顶部带有品牌色至深蓝色的渐变 Header。
- **交互**: 对话气泡区分左右。用户气泡使用主色实心；AI 气泡使用白底带微阴影。支持在对话中插入高亮的“建议行动”小卡片。

## 四、 页面模板 (Page Templates)

页面模板是基于上述 Token、语义和组件组装而成的标准化页面级解决方案，确保不同终端下提供专业且一致的用户体验。

### 1. 列表模板 (List Template)
*适用场景：设备台账、工单列表、告警历史、巡检任务。*
- **页面头部**: 左侧“大标题 + 描述”，右侧核心操作区（主要操作使用 `#0244FD`，次要操作使用带边框的白底按钮）。
- **筛选区 (Filter Bar)**: 紧贴表格上方，背景采用比纯白更浅的 `slate-50/50`，包含搜索输入框和高级筛选按钮。
- **分页器 (Pagination)**: 置于表格底部，当前页码按钮使用品牌色 `#0244FD` 实心填充。

### 2. 详情模板 (Detail Template)
*适用场景：设备台账详情、工单进度详情、告警处理视图。*
- **布局结构**: 采用非对称网格（如 7:3 或 8:4）。左侧为主内容区（Main Column），右侧为辅助/元数据区（Sidebar Column）。
- **左侧主区**: 基础信息采用 `Key-Value` 键值对网格展示；核心进度使用“垂直时间轴”组件。
- **右侧辅助区**: 放置 AI 助手面板、关系图谱树或相关附件列表。

### 3. 表单模板 (Form Template)
*适用场景：新建维保计划、注册新设备、填写巡检反馈。*
- **卡片化分割**: 长表单必须进行物理分割（如：基础信息、技术参数、维保策略三张卡片），并带上小标题。
- **底部操作栏**: 长表单底部需提供固定在视窗底部的操作条（带顶部阴影和模糊背景 `backdrop-blur`），确保提交入口始终可见。

### 4. 分析模板 (Analysis Template)
*适用场景：监控大屏 (Dashboard)、能耗深度分析、设备健康度看板。*
- **布局网格**: 采用 Bento Box（便当盒）网格布局，卡片间距统一 `16px` 或 `24px`。
- **图表规范**: 图表主线条/柱体必须使用品牌色 `#0244FD` 或其降级色。严禁滥用超过 3 种高饱和度颜色。必须配备悬浮提示框（Tooltip，背景深色）。
- **无数据态**: 必须展示占位卡片（虚线边框 + 轻量级插画 + 提示文案），不可留白。
